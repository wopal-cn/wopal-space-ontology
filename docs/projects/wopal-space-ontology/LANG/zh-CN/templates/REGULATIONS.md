# REGULATIONS.md — 空间守则

本法规是本工作空间内的根本行为规范。所有 Agent **必须**严格遵守。

## 第一部分：操作和安全红线

<CRITICAL_RULE>

### 误删防护

- 未经审慎考量不得删除任何文件或目录
- **授权要件**：仅当用户以命令式语句（如"删掉它"）明确确认时，方可执行删除
- 疑问形式（如"是否应该删除"）不构成授权
- **删除前必读**：任何删除操作前必须先读取文件内容，判断是否为用户工作产物
- **未跟踪文件保护**：`git status` 显示 `??`（未跟踪）的文件，一律禁止删除
- 删除目录必须用 `trash`，禁止 `rm -fr`
- 删除文件必须用 `trash`，禁止 `rm`
- `git rm` 同样删除磁盘文件，同属误删防护

### 工作边界

- 操作领域限定为空间根目录
- 未经授权不得越出此边界修改系统配置或隐私文件
- 代码或提交中不得包含 `.env`、密钥等敏感凭证

### 目录保护

- `external/` 是外部资源引用集合
- 未经授权不得创建、修改或删除 `external/` 下任何内容

### 技能管理

技能位于 `.wopal/skills/` 目录，可直接编辑。

| 操作 | 方式 |
|------|------|
| 编辑已有技能 | 直接编辑 `.wopal/skills/<name>/` |
| 安装外部技能 | `wopal skills install <url/path>` |
| 删除技能 | 删除 `.wopal/skills/<name>/` 目录 |

### Git 历史不可变原则

已提交的代码就是事实，只能向前修正，不能向后抹杀。

**绝对禁止**：

| 操作 | 原因 |
|------|------|
| `git reset --hard <非HEAD>` | 丢弃提交历史 |
| `git reset --soft <非HEAD>` | 回退提交历史 |
| `git commit --amend`（已 push 的 commit）| 改写已推送历史，补救必须 force push |
| `git checkout <file>` / `git restore <file>`（未 diff 确认前）| 覆盖未提交变更，等同于删除工作成果 |

**修正策略**（按优先级选择）：

1. **常规修正**：新 commit 直接提交（最安全，优先使用）
2. **精确编辑**：用 `edit` 工具精确删除错误部分，保留正确修改
3. **rebase**（受严格限制）：仅在满足所有条件时使用 — commit 未推送、本会话创建、仅修正误带文件、用户明确授权

</CRITICAL_RULE>

## 第二部分：工作规范

### 核心技能

每个 WopalSpace 都提供以下核心技能。Agent 应在相应场景下主动调用。

| 技能 | 空间职责 | 触发场景 |
|------|---------|---------|
| `space-master` | 空间技能根与流程路由总入口 | 任务意图不清、空间运维、ontology 协作、技能体系、流程选择、多 Space 管理 |
| `agents-collab` | 子代理协作协议 | 任何 fae、rook 或 general 子代理委派前 |
| `dev-flow` | Issue/Plan 驱动开发状态机 | Issue、Plan、审批、执行、验证、归档 |

### 技能调用

- 执行前检查上下文中的 `<available_skills>` 列表
- 新建/修改技能需遵循 `skill-creator` 技能指导
- **严禁**：放着现成技能不用，强行用通用能力执行

### Git 工作流基础

- **实施前检查**：开始前运行 `git status` 和 `git log --oneline -5`
- **用户确认门禁**：所有代码变更必须通过用户验证后才能提交
- **提交前检查**：运行 `git status`、`git diff --staged --name-only`、`git log --oneline -5`
- **提交格式**：`<type>(scope): <description> [#ref]`

### 子代理委托

<CRITICAL_RULE>

委派**任何** subagent 前：
1. 搜索记忆中的委派规则和过往教训
2. 检查 prompt 中所有路径使用空间根目录相对路径或绝对路径
3. 确认 prompt 中包含目标项目路径上下文

</CRITICAL_RULE>

### 验证隔离

验证和测试工作不得污染空间项目文件。验证操作只能在 `.wopal-space/.tmp/` 或系统临时文件夹中进行。

### 记忆与进化

记忆只有被主动检索才有价值。以下场景必须主动搜索记忆：

| 场景 | 搜索关键词 |
|------|-----------|
| 复杂任务开始前 | 任务类型关键词 |
| 遇到模糊/冲突指令 | 相关主题关键词 |
| 用户批评后 | 问题领域关键词 |
| 关键决策节点 | 该节点关键词 |
| 工具执行错误后 | 任务类型关键词 |

**记忆写入规则**：
- 长期记忆写入需：先去重 → 展示完整内容给用户 → 等待明确确认 → 执行写入
- 仅记录与空间优化、项目建设高度相关且具有长期复用价值的信息
- 记忆与 AGENTS.md / REGULATIONS 冲突时 → 规范优先；记忆有独特细节 → 合并到规范后删除记忆

## 第三部分：工程细节

### 时间处理

- 必须用系统命令获取时间，严禁推断
- 命令：`date '+%Y-%m-%d %H:%M:%S'`

### 路径确认

- 未明确路径的文件，必须先用 `glob` 确认结构，禁止假设路径

### 工程规范

- **历史文档不追溯**：`docs/projects/**/plans/done/` 等历史记录不追溯修改
