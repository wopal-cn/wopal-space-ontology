# 空间使用指南 — 通用参考

## 协作方式

在这个空间里，你可以通过三种方式和我协作：

**1. 直接和 Agent 对话。**
告诉我你想做什么，比如"帮我创建一个项目"、"review 这段代码"、"看看这个 Issue"。我会自动判断加载什么技能、是自己处理还是委派给子 Agent。

**2. 用 slash 命令执行常见操作。**
常用操作的快捷入口（见下方"核心命令"章节）。Slash 命令在 ellamaka 会话中使用。

**3. 用 wopal CLI 做系统级操作。**
查看空间状态、管理本体等确定性操作，在终端执行 `wopal` 命令。

简单原则：**语义工作找 Agent，系统操作用 CLI。**

## 重要文件和目录

| 位置 | 是什么 | 什么时候用到 |
|------|--------|-----------|
| `projects/` | 代码项目 | clone 仓库、新建项目 |
| `contents/` | 内容创作 | 写博客、教程 |
| `docs/` | 跨项目文档 | 产品文档、设计文档 |
| `AGENTS.md` | 空间入口 | 给 Agent 加自己的规则 |
| `.wopal-space/STRUCTURE.md` | 空间结构索引 | Agent 用它了解空间全貌 |
| `.wopal-space/REGULATIONS.md` | 空间守则 | 查看或修改 Agent 行为规则 |
| `.wopal-space/memory/USER.md` | 用户画像 | Agent 每次启动都会读 |

---

## 核心命令

| 命令 | 做什么 | 什么时候用 |
|------|--------|-----------|
| `/help` | 查看空间使用方法 | 忘了怎么用的时候 |
| `/init` | 校准空间结构 | 加了新项目/目录后更新 STRUCTURE.md |
| `/commit` | 按规范格式提交代码 | 代码改完要提交 |
| `/review` | 审查代码质量 | 想让 Agent 检查代码 |
| `wopal space status` | 查看空间与类型的同步状态 | 想知道空间是否落后 |
| `wopal ontology status` | 查看本体源的同步状态 | 想知道上游是否有更新 |

---

## 核心技能

技能是 Agent 的工作流模块。它们会自动加载，你不需要手动触发。

| 技能 | 管什么 | 什么时候加载 |
|------|--------|-----------|
| `space-master` | 流程路由、空间运维、本体协作 | Agent 不确定该怎么做时 |
| `agents-collab` | 子代理协作协议 | 任何 fae/rook 委派前 |
| `dev-flow` | Issue/Plan 开发状态机 | 创建 Issue、推进 Plan 时 |

---

## 守则与自定义

**规则在哪：**
- `.wopal-space/REGULATIONS.md` — Agent 必须遵守的守则（安全红线、Git 规范、子代理协作）
- `AGENTS.md` — 用户空间个性化规则入口

**想改 Agent 行为怎么办：**
1. 改 `REGULATIONS.md` 调整守则
2. 改 `AGENTS.md` 加你自己的规则
3. 或者直接告诉我"记住我的偏好"，我会更新 `USER.md`

你不需要现在就读懂所有规则。知道位置就够了，想改的时候告诉我。
